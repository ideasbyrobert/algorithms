Merge Sorted Array · Optimal Solution

Fill from the back. No overwrites.

Given two sorted arrays where the first has enough trailing space to hold both, merge them in-place into one sorted array. The key insight: fill from the tail to avoid overwriting unread elements.

Why Backward?

Forward merging would write into positions that the primary read pointer hasn't reached yet — corrupting data. By starting at the last position and placing the largest remaining value each step, the write pointer always lands in empty trailing space.

Algorithm

Algorithm

p ← m - 1,  s ← n - 1,  w ← m + n - 1
        while s ≥ 0:
        if p ≥ 0 and primary[p] > secondary[s]:
        primary[w] = primary[p]
        p -= 1
        else:
        primary[w] = secondary[s]
        s -= 1
        w -= 1

Edge Patterns

Secondary exhausted first — remaining primary elements are already sorted in place. No further work needed.

Primary exhausted first — remaining secondary elements fill the front positions without comparison.

All elements from one side — one pointer sweeps while the other stays put. Still O(m+n) total steps.

Complexity

Complexity

Time: O(m + n) — each element placed exactly once.
Space: O(1) — merge happens in-place in the primary array.
