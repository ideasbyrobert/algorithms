Rotate Array

LeetCode 189 · Three-Reversal Trick

Rotate an array to the right by k steps. Instead of shifting elements one at a time (O(n·k)), reverse three segments in O(n) total — the whole array, the front k, then the back n−k.

Three Reversals

Step 1: Reverse the entire array — this puts the tail block (which should move to front) into position, but both blocks are backwards.
Step 2: Reverse the first k elements — fixes the front block.
Step 3: Reverse the remaining n−k elements — fixes the back block.

Algorithm

k ← k % n
reverse(0, n - 1)     // whole array
reverse(0, k - 1)     // front block
reverse(k, n - 1)     // back block

Edge Patterns

k ≥ n — normalize: k = k % n. Rotating by the full length is a no-op.

k = 0 — array unchanged; skip all reversals.

Each reversal is a two-pointer swap from endpoints inward — O(segment length/2) swaps.

Complexity

Time: O(n) — each element touched at most twice.
Space: O(1) — in-place swaps.
