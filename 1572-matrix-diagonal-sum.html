Matrix Diagonal Sum · Optimal Solution

Two diagonals, one pass, one sum.

Given a square matrix, return the sum of both diagonals. The primary diagonal runs top-left to bottom-right. The secondary diagonal runs top-right to bottom-left. When the matrix has odd size, the center cell sits on both diagonals and must be counted only once.

Concept

Two Diagonals, One Overlap

The primary diagonal visits matrix[r][r] — the row and column indices always match. The secondary diagonal visits matrix[r][n − 1 − r] — the column counts down as the row counts up. In an odd-sized matrix both diagonals cross at the center cell matrix[n/2][n/2], which would be double-counted if left uncorrected.

Algorithm

Sum Then Correct

Walk every row once. For each row, add the primary diagonal cell and the secondary diagonal cell to a running sum. After all rows, subtract the center value if the matrix is odd-sized. 

sum ← 0
        for r in 0 .. n-1:
        sum ← sum + matrix[r][r]
        sum ← sum + matrix[r][n-1-r]
        if n is odd:
        sum ← sum − matrix[n/2][n/2]
        return sum
Edge Cases

Edge Patterns

Even-sized matrix — the two diagonals never share a cell, so no correction is needed.
Single cell — both diagonals contain only that cell; the correction removes the duplicate and the answer is the cell value itself.
Large center — when the center value is large, failing to subtract it produces a visibly wrong answer, making the guard condition easy to verify.

Complexity

Complexity

Time: O(n) — one pass over the rows.
Space: O(1) — only a running sum.
