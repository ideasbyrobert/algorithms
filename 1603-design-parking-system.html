Design Parking System · Optimal Solution

Three counters. One decision per car.

Design a parking system with three types of spaces — big, medium, and small. Each type has a fixed capacity. When a car arrives, it parks in its matching type or is rejected if that type is full.

Concept

Independent Buckets

Each car type maps to exactly one counter. A big car cannot take a medium slot. The system is three independent decrement-or-reject counters — no cross-type borrowing.

The common mistake is using a boolean "has space" flag instead of an integer count. A boolean forgets that capacity can be greater than one.

Edge Cases

Edge Patterns

Zero initial capacity — A type initialized to 0 rejects every car immediately. No decrement occurs.

Exhaustion doesn't affect others — Filling all big slots leaves medium and small unchanged.

Repeated same-type — Each arrival decrements the counter by one until it reaches zero.

Complexity

Complexity

Time: O(1) per AddCar — single array lookup and decrement.
Space: O(1) — three integer counters.
