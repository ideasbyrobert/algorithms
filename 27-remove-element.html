Remove Element

LeetCode 27 · Write Pointer Compaction

Given an array and a value, remove all instances of that value in-place and return the new length. The order of remaining elements may change.

Read/Write Pointer

A read pointer scans every element. A write pointer tracks where the next keeper should land. When the read pointer finds a non-target value, it copies it to the write position and both advance. When it finds the target, only the read pointer moves — the write pointer stays put, effectively skipping the match.

Algorithm

write ← 0
for read in 0..n-1:
    if nums[read] ≠ val:
        nums[write] = nums[read]
        write += 1
return write

Edge Patterns

No matches — write tracks read exactly; every element is copied to itself. The array is unchanged.

All matches — write never advances; the result length is 0.

Write ≤ read invariant — the write pointer never overtakes the read pointer, so no unread data is overwritten.

Complexity

Time: O(n) — single pass.
Space: O(1) — in-place with two pointers.
