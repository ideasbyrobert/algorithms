Majority Element

LeetCode 169 · Boyer-Moore Voting Algorithm

Find the element that appears more than ⌊n/2⌋ times. The voting algorithm needs only one pass and O(1) space — no hash map, no sorting.

Candidate and Balance

Maintain a candidate and a balance counter. When balance hits zero, adopt the current value as the new candidate. Matching values reinforce (balance++); mismatches cancel (balance−−). The true majority always survives because it can't be fully cancelled.

Algorithm

candidate ← null, balance ← 0
for each value:
    if balance == 0:
        candidate ← value, balance ← 1
    else if value == candidate:
        balance += 1
    else:
        balance -= 1
return candidate

Edge Patterns

Immediate restart — when balance drops to zero mid-array, the next element becomes the new candidate.

Uniform array — balance simply increments every step; no cancellation occurs.

Guarantee — Boyer-Moore only works when a majority is guaranteed to exist. Without that guarantee, a verification pass is needed.

Complexity

Time: O(n) — single pass.
Space: O(1) — two variables.
